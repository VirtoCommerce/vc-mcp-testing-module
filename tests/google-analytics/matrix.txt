## 1. `view_item_list` (function: `viewItemList`)

### Parameters

| Param         | Source                            | Type   | Notes                    |
| ------------- | --------------------------------- | ------ | ------------------------ |
| `items_skus`  | `toCSV(items.map(el => el.code))` | string | CSV of item codes        |
| `items_count` | `items?.length ?? 0`              | number | Total items in list      |
| `...params`   | from function arg                 | any    | generic tracking context |

### Test ideas

**Positive**

1. List with N items → `items_count = N`, `items_skus` contains all codes, no extra.
2. Empty list (`[]`) → `items_count = 0`, `items_skus = ""` (or undefined as per `toCSV` behavior).

**Negative**

1. `items = null` or `undefined` → event still sent, `items_count = 0` without runtime error.
2. Some items missing `.code` → verify behavior (either `""` for that item or it’s excluded – define expected behavior with dev).

**Edge**

1. Very large list (e.g. 100+ items) → event still sent, `items_skus` not truncated or broken.
2. Item codes with special characters (spaces, commas) → ensure CSV and decoding work.
